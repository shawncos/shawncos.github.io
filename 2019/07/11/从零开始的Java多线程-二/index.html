从零开始的Java多线程(二)

发表于 2019-07-11 分类于 -Java 阅读次数：

Thread类一共有四个构造方法，都会调用Thread类的init()方法。
这里有一些常用方法。
1.static Thread currentThread(); 返回当前线程的引用。
2.long getId();返回线程的标识符。
3.String getName();返回此线程的名称。
4.int getPriority();返回此线程的优先级。
5.boolean isAlive();测试此线程是否处于活动状态。(活动状态就是线程已经启动且尚未终止。线程处于正在运行或准备运行的状态。)
6.static void sleep(long millis);使当前正在执行的线程以指定的毫秒数暂停（暂时停止执行），具体取决于系统定时器和调度程序的精度和准确性.
7.void interrupt();中断这个线程。
8.static boolean interrupted();测试此线程是否中断，执行后具有将状态标志清除为false的功能。
9.boolean isInterrupted();测试此线程是否中断，不清除状态。
10.void setName(String name);设置此线程的名字。
11.boolean isDaemon();是否为守护线程。（守护线程拥有自动结束自己生命周期的特性，而非守护线程不具备这个特点。）
12.boolean setDaemon(boolean on);将此线程标记为daemon线程或用户线程。
13.void join();等待线程终止。join()的作用是：“等待该线程终止”，这里需要理解的就是该线程是指的主线程等待子线程的终止。也就是在子线程调用了join()方法后面的代码，只有等到子线程结束了才能执行。
14.static void yield();放弃cpu资源，让给其他线程，放弃的时间不确定，可能一会就会重新获得CPU时间片。
15.void setPriority(int newPriority);设置线程的优先级。

原来有suspend ,stop,resume 方法有死锁倾向。interput方法可以终止线程

运行以上代码，发现并不能终止线程。
注释的部分，interrupted(),return之后终止线程。
